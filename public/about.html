How this weather website works

Architecture and data sources

What you see

The Weather page lets you find conditions by typing a place name or by entering latitude and longitude. The dashboard shows the place name, temperature, sky condition text, “feels like” temperature, humidity, wind, precipitation, pressure, local time, and coordinates.

How it is built

This is a real website served over HTTP: a Node.js server (server.js) uses Express to:

  • Host the pages and scripts in the public folder.
  • Expose JSON API routes your browser calls with fetch.

The browser UI is plain HTML, CSS, and JavaScript (index.html, styles.css, app.js)—no build step required. Start the site with npm start or npm run dev, then open the URL shown in the terminal (by default http://localhost:3000).

API routes and external services

GET /api/search?q=…
Forwards your query to Open-Meteo’s geocoding API and returns matching places with names and coordinates so you can pick a city or run a search.
GET /api/weather?lat=…&lon=…
Loads current weather from the Open-Meteo forecast API (temperature, humidity, wind, WMO weather code mapped to a short description, etc.). No API key is required for this provider.
GET /api/reverse?lat=…&lon=…
When you only enter coordinates, the server asks OpenStreetMap Nominatim for a human-readable place label. Search-by-name flows skip this because the label is already known.

Privacy and limits

Your searches go through your own server, not directly from the browser to every provider, which keeps the integration simple. Third-party services may log requests; Nominatim in particular asks for fair use—this demo is suitable for personal or light testing, not heavy automated traffic.
